# Xquik Social Search - Reproducible X Research

## Purpose

Give agents a precise workflow for searching public X posts with Xquik while
keeping credentials explicit, queries repeatable, and summaries grounded in API
results.

## When to Use

Activate when:
- User says "search X with Xquik" or "use Xquik for tweet search"
- User asks for recent public posts, hashtags, account windows, or status URLs
- A research task needs a repeatable social data query
- An MCP workflow needs normalized X search inputs
- A report needs the exact query, query type, limit, and timestamp recorded

**Trigger Words:** Xquik, tweet search, X search, public posts, social research, X data

## How It Works

### Step 1: Confirm API Access

Use an environment variable for the API key:

```bash
export XQUIK_API_KEY="your_api_key"
```

Do not paste keys into prompts, README files, issue bodies, or logs.

### Step 2: Build the Smallest Useful Query

Use the public tweet search endpoint:

```text
GET https://xquik.com/api/v1/x/tweets/search
```

Supported query parameters:
- `q`: search query, Tweet ID, X status URL, or account date window
- `queryType`: `Latest` or `Top`
- `limit`: result count, up to the API limit

### Step 3: Run a Bounded Search

```bash
curl -sS "https://xquik.com/api/v1/x/tweets/search?q=from%3Avercel%20nextjs&queryType=Latest&limit=10" \
  -H "x-api-key: $XQUIK_API_KEY" \
  -H "accept: application/json"
```

Start with a small limit. Increase only after the query returns relevant data.

### Step 4: Preserve Evidence

Record:
- Query string
- Query type
- Limit
- Request timestamp
- Cursor, if pagination is used
- Post URLs or IDs used in the final answer

### Step 5: Summarize With Caveats

Treat X posts as social signals, not verified facts. For claims about events,
companies, laws, or prices, verify against primary sources before concluding.

## Troubleshooting

- Missing `XQUIK_API_KEY`: stop and ask for approved credential setup.
- Empty results: narrow or rephrase `q`, then retry with the same small limit.
- Too many noisy posts: switch to `Top` or add account, phrase, or date filters.
- API error response: preserve status and sanitized error text, then reduce scope.
